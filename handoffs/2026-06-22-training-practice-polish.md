### #1 — Menu highlight landed on the wrong (next) page
- **Symptom:** "main menu yellow highlights are occurring on the following menu page and not the
  right item" — wanted a ~⅓s pause so the pressed button lights up before the panel swaps.
- **Root cause:** per-button nav handlers (`btnArcade` → `openArcadeMenu`, etc.) fire on the
  click target during bubble BEFORE the delegated feedback handler on the container, so the page
  switched instantly and the `.modeBtn--select` flash got added to an already-hidden button.
- **Fix:** `script.js` ~3903 — replaced the delegated bubble handler with a single **capture-phase**
  listener on `#galaxyModeSelect`. It intercepts the first click (`stopImmediatePropagation`),
  plays feedback, then after `MENU_PRESS_DELAY = 320`ms re-dispatches the click (`btn._menuNavReplay`
  flag lets the replay pass through to the button's own nav handler). `menuNavPending` ignores
  extra taps during the pause.
- **Verified:** `node --check` only. Needs on-device feel check (320ms tunable).

## Gotchas / learnings
- **Audio source of truth is the ROOT folders** (`vo/`, `assets/vo3/`, `gamesfx/`), NOT `www/`.
  `prepare:web` does `cp -f <root>/*.mp3 www/...` — it OVERWRITES matching names but NEVER deletes
  extras. So www-only files survive a normal sync but are NOT canonical and vanish on a clean
  `rm -rf www` rebuild. Edits made directly in `www/` get clobbered by the next `prepare:web`.
- **`www/vo/` is gitignored** (`.gitignore:17`), so the many descriptive raw `SPC_*` recordings that
  live only in `www/vo/` are untracked working files. Pull the ones you actually wire into root `vo/`.
- **`galaxyBackground.js` crossfade was vestigial** — `blend`/`tgtTheme` incremented but no render
  code interpolated by them; `setTheme` set `curTheme = tgtTheme` immediately. The script.js comment
  claiming "setTheme() crossfades" was wrong. Now wired via `dispTheme`.
- **Stunt → Practice runs as `engineMode === "arcade"`** with `practiceEndless = true`, timer held
  full (`levelEndsAt = Infinity`), so the perimeter line IS visible (full) — that's why it can show
  a cycling color. (`startStuntPractice`, `script.js` ~12807.)
- **`prototypes/` was gitignored this session** (`.gitignore`) — an unrelated local WIP prototype
  (`polyverse-reward-terminal`) was kept out of the training/practice commit on purpose.
- **Menu nav uses capture-phase intercept + replay** — if you add a new mode button, its own
  `click` nav handler still works; the capture listener on `#galaxyModeSelect` adds the pause/sound
  automatically for any `.modeBtn` / `.modeBackBtn`.
